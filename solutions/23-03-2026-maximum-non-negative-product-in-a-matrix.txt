Problem: Maximum Non-negative Product in a Matrix

Approach:
We are given a grid where we can only move right or down, and we need to find the maximum product from (0,0) to (m-1,n-1).

The tricky part is that the grid can contain negative numbers. A negative value can turn a small product into a large one (if multiplied again), so we must track both:

maximum product so far & minimum product so far

For each cell (i, j), we store:
{maxProduct, minProduct} from that cell to the end.

We use recursion + memoization (DP):

From (i, j), we try moving down and right
For each direction, we get both max and min values
We multiply the current cell value with both and update:
new max = max of all possibilities
new min = min of all possibilities

We store results in a DP table so we don’t recompute the same cell again.

At the end:

If the maximum product is negative, return -1
Otherwise, return it modulo 
1e9+7

Time Complexity: O(m × n)
Space Complexity: O(m × n)


```cpp

class Solution {
public:

    int n, m;
    typedef long long ll;
    int M = 1e9+7;

    vector<vector<pair<ll, ll>>> t;

    pair<ll, ll> solve(int i, int j, vector<vector<int>>& grid)
    {
        if (i == m-1 && j == n-1) return {grid[i][j], grid[i][j]};

        ll maxVal = LLONG_MIN;
        ll minVal = LLONG_MAX;

        if (t[i][j] != make_pair(LLONG_MIN, LLONG_MAX))
        {
            return t[i][j];
        }

        // for down
        if (i+1 < m)
        {
            auto [downMax, downMin] = solve(i+1, j, grid);
            maxVal = max({maxVal, grid[i][j] * downMax, grid[i][j] * downMin});
            minVal = min({minVal, grid[i][j] * downMax, grid[i][j] * downMin});
        }

        // for right
        if (j+1 < n)
        {
            auto [rightMax, rightMin] = solve(i, j+1, grid);
            maxVal = max({maxVal, grid[i][j] * rightMax, grid[i][j] * rightMin});
            minVal = min({minVal, grid[i][j] * rightMax, grid[i][j] * rightMin});
        }

        return t[i][j] = {maxVal, minVal};
    }

    int maxProductPath(vector<vector<int>>& grid) {

        m = grid.size();
        n = grid[0].size();
        t = vector<vector<pair<ll ,ll>>>(m, vector<pair<ll, ll>>(n, {LLONG_MIN, LLONG_MAX}));

        auto [maxProd, minProd] = solve(0, 0, grid);

        return maxProd < 0 ? -1 : maxProd % M;
        
    }
};
