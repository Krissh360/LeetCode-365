Problem: Determine Whether Matrix Can Be Obtained by Rotation

Approach:
We are given two matrices, mat and target. We need to check if mat can become target after rotating it by 0°, 90°, 180°, or 270°.

Instead of writing separate logic for each rotation, we use a simple trick:
Rotate the matrix 90° at a time
After each rotation, check if it matches the target

To rotate a matrix by 90° clockwise:
Transpose the matrix (swap rows and columns)
Reverse each row

This effectively rotates the matrix in-place. We repeat this process up to 4 times

Before each rotation, we check if mat == target. If at any point they match, we return true.
Otherwise, we rotate and try again

If none of the 4 rotations match, we return false

Time Complexity: O(n^2)
Space Complexity: O(1)


```cpp

class Solution {
public:

    void rotate(vector<vector<int>>& mat, int n)
    {
        // Transpose
        for (int i = 0; i < n; i++)
        {
            for (int j = i; j < n; j++)
            {
                swap(mat[i][j], mat[j][i]);
            } 
        }

        // Reverse
        for (int i = 0; i < n; i++)
        {
            reverse(mat[i].begin(), mat[i].end());
        }
    }

    bool findRotation(vector<vector<int>>& mat, vector<vector<int>>& target) {
        int n = mat.size();

        for (int c = 0; c < 4; c++)
        {
            bool equal = true;

            for (int i = 0; i < n; i++)
            {
                for (int j = 0; j < n; j++)
                {
                    if (mat[i][j] != target[i][j])
                    {
                        equal = false;
                        break;
                    }
                }

                if (!equal) break;
            }

            if (equal)
                return true;
            rotate(mat, n);
        }

        return false;
        
    }
};
