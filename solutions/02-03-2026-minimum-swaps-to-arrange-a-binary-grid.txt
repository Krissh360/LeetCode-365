Problem: Minimum Swaps to Arrange a Binary Grid

Approach:
We are given an n x n binary grid. The goal is to rearrange the rows using the minimum number of adjacent swaps so that for every row i, there are at least n - i - 1 trailing zeros.

The key idea is to first compute how many zeros appear at the end of each row. For every row, we start from the last column and count consecutive zeros moving left. We store this count in a separate array.

Now, for row i, we need at least n - i - 1 trailing zeros. So we try to find a row at or below index i that satisfies this condition.

If we find such a row at index j, we bring it up to position i using adjacent swaps. Each swap moves the row up by one position, so the number of swaps needed is j - i. We simulate this by shifting values in our trailing zero array.

If no such row exists, it means arranging the grid as required is impossible, so we return -1.

We repeat this process for each row.

Time Complexity: O(n²)
Space Complexity: O(n)


```cppclass Solution {
public:
    int minSwaps(vector<vector<int>>& grid) {

        int n = grid.size();
        vector <int> endZero(n, 0);
        
        for (int i = 0; i < n; i++)
        {
            int j = n-1, count = 0;

            while (j >= 0 && grid[i][j] == 0)
            {
                count++;
                j--;
            }

            endZero[i] = count;
        }  

        int steps = 0;

        for (int i = 0; i < n; i++)
        {
            int need = n-i-1, j = i;

            while (j < n && endZero[j] < need) j++;

            if (j == n) return -1;

            steps += j-i;

            while (j > i)
            {
                swap(endZero[j], endZero[j-1]);
                j--;
            }
        }

        return steps;
    }
};
